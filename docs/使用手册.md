## 1. 这是什么

**Cursor Agent Status** 是一个常驻 macOS 菜单栏的小工具。当你在 Cursor 里使用 Agent（Composer Agent 模式）时，它会在后台监听 Cursor Hooks 上报的事件，并在界面上告诉你：

- 当前有多少任务**正在进行**
- 有哪些操作**等待你确认**
- 哪些任务**刚刚完成**

应用包含两种展示方式：

| 方式 | 说明 |
|------|------|
| **菜单栏图标** | 常驻屏幕右上角，点击展开简要面板 |
| **悬浮窗** | 更大、可置顶的面板，展示完整任务列表 |

> 应用不会显示在 Dock 栏，这是正常设计（菜单栏专用应用）。

## 2. 系统要求

- **操作系统**：macOS 14.0 或更高
- **Cursor**：已安装并可正常使用 Agent 功能
- **依赖**：Python 3（系统自带即可，用于 Hook 脚本）
- **可选**：Xcode（用于自行编译应用）

## 3. 安装

安装分两步：**先装 Hooks，再启动 Mac 应用**。Hooks 负责采集 Cursor 事件，Mac 应用负责展示。

### 3.1 安装 Hooks（只需一次）

在终端执行：

```bash
cd ~/Projects/cursor-agent-status
chmod +x scripts/install-hooks.sh hooks/status-bridge.sh hooks/status-bridge.py
./scripts/install-hooks.sh
```

脚本会：

1. 将 `status-bridge.sh` 和 `status-bridge.py` 复制到 `~/.cursor/hooks/`
2. 安装或合并 `~/.cursor/hooks.json`（若已有 hooks 会先备份）
3. 创建数据目录 `~/.cursor/agent-status/`

**安装后请重启 Cursor**，或保存 `~/.cursor/hooks.json` 以触发重新加载。

### 3.2 构建 Mac 应用

**方式 A：Xcode（推荐）**

```bash
open ~/Projects/cursor-agent-status/CursorAgentStatus.xcodeproj
```

在 Xcode 中按 **⌘R** 运行，或 **Product → Archive** 打包安装。

**方式 B：命令行构建**

```bash
cd ~/Projects/cursor-agent-status
./scripts/generate-xcodeproj.sh
xcodebuild -project CursorAgentStatus.xcodeproj \
  -scheme CursorAgentStatus \
  -configuration Release \
  build
```

构建完成后，在 DerivedData 中找到 `CursorAgentStatus.app` 并打开，或拖入「应用程序」文件夹。

**方式 C：验证安装是否完整**

```bash
./scripts/verify-integration.sh
```

全部显示 `OK` 即表示 Hooks 和应用均可正常工作。

## 4. 启动应用

### 启动方式

```bash
# 若已构建 Debug 版
open ~/Library/Developer/Xcode/DerivedData/CursorAgentStatus-*/Build/Products/Debug/CursorAgentStatus.app

# 或从「应用程序」文件夹双击打开
```

### 启动后

1. 菜单栏右上角会出现 **✨ / 🔄 / ✋** 图标（见下文状态说明）
2. **不会**出现在 Dock 栏
3. 若图标被折叠，点击菜单栏右侧的 `>>` 查找

### 开机自启（可选）

将 `CursorAgentStatus.app` 拖入：

**系统设置 → 通用 → 登录项 → 打开 at 登录**

## 5. 界面说明

### 5.1 菜单栏图标

| 图标 | 含义 |
|------|------|
| ✨ `sparkles` | 空闲，无进行中的任务 |
| 🔄 `arrow.triangle.2.circlepath` | 有任务正在进行 |
| ✋ `hand.raised` | 有待确认的操作 |

图标右上角**红色数字**表示「进行中」任务数量（最多显示 99）。

### 5.2 菜单栏下拉面板

点击菜单栏图标展开，包含：

**顶部统计卡片**

- **进行中**（蓝色）：当前活跃任务数
- **待确认**（橙色）：等待批准或输入的任务数
- **刚完成**（绿色）：最近 60 秒内完成的任务数

**三个任务列表**（各最多显示 5 条）

- 点击任意任务可尝试打开对应的 transcript 文件或 Cursor

**底部操作按钮**

| 按钮 | 功能 |
|------|------|
| 打开 Cursor | 启动 Cursor 应用 |
| 显示/隐藏悬浮窗 | 切换详细面板 |
| 重置状态 | 清除所有进行中和待确认任务（见 FAQ） |
| 退出 | 关闭本应用 |

### 5.3 悬浮窗

点击「显示悬浮窗」打开，提供：

- 完整的三个分区任务列表（不限 5 条）
- **始终置顶**：窗口保持在最上层
- **空闲时自动隐藏**：无任务时自动收起窗口
- **透明度**：调节窗口透明度（0.6 ~ 1.0）
- **重置状态**：同菜单栏

悬浮窗可拖动，关闭按钮会隐藏窗口（不会退出应用）。

## 6. 三种任务状态

### 进行中（蓝色）

表示 AI 当前正在执行的工作，包括：

| 类型 | 触发时机 |
|------|----------|
| Agent 会话 | 新建 Composer 对话 |
| 工具执行 | Read、Write、Shell、Grep 等工具调用 |
| Subagent | Task 工具启动的子代理 |

### 待确认（橙色）

表示需要你介入的操作：

| 类型 | 说明 |
|------|------|
| Shell 命令 | Agent 即将执行终端命令，等待批准 |
| MCP 工具 | Agent 即将调用 MCP 工具，等待批准 |
| 等待用户输入 | Agent 已回复，等待你发送下一条消息 |
| 操作被拒绝 | 你拒绝了某次操作，Agent 可能需要重新处理 |

> **注意**：Cursor Smart Mode 内置的审批界面不一定能被 Hook 捕获，部分待确认状态可能无法显示。

### 刚完成（绿色）

最近 **60 秒内**结束的任务，包括：

- Subagent 完成
- Agent 一轮回复结束
- Agent 任务循环结束（`stop`）
- 会话正常关闭

60 秒后自动从列表中淡出消失。

## 7. 日常使用

### 典型工作流

```
1. 启动 Cursor Agent Status（菜单栏图标出现）
2. 在 Cursor 中打开 Agent，发送任务
3. 观察菜单栏角标数字增加 → AI 开始工作
4. 若有 Shell/MCP 需批准 → 「待确认」计数增加
5. Agent 回复完成 → 「刚完成」出现新条目
6. 点击「显示悬浮窗」查看详细列表
```

### 自动过期机制

为避免 Cursor 异常退出后残留「假进行中」任务，应用会自动清理：

| 类型 | 过期时间 |
|------|----------|
| 工具 / Subagent | 90 秒无更新 |
| Shell/MCP 待确认 | 60 秒无更新 |
| 会话 | 3 分钟无任何新事件 |

### 手动重置

若重启 Cursor 后仍看到旧的进行中任务，点击 **「重置状态」** 即可立即清空。

## 8. 常见问题

### Q：菜单栏找不到图标？

- 检查应用是否在运行：`pgrep -l CursorAgentStatus`
- 查看菜单栏右侧 `>>` 折叠区
- 重新启动应用

### Q：Cursor 里 Agent 在工作，但应用显示 0？

1. 确认 Hooks 已安装：`cat ~/.cursor/hooks.json | grep status-bridge`
2. 重启 Cursor
3. 检查事件文件是否有写入：
   ```bash
   tail -f ~/.cursor/agent-status/events.jsonl
   ```
   在 Cursor 中触发 Agent 操作，应看到新行出现
4. 重启 Cursor Agent Status 应用

### Q：重启 Cursor 后仍显示很多「进行中」？

这是 Cursor 重启时未发送 `sessionEnd` 事件导致的残留。解决方法：

1. 点击 **「重置状态」**
2. 或等待 1~3 分钟，过期机制会自动清理
3. 重新启动 Cursor Agent Status 应用（已内置启动时清理）

### Q：Cloud Agent（cursor.com/agents）的状态能显示吗？

**不能。** 用户级 Hooks 仅作用于本地 Cursor IDE，Cloud Agent 运行在远程 VM，不会触发 `~/.cursor/hooks.json`。

### Q：待确认数量不准？

Hook 能稳定捕获 Shell/MCP 执行前后的时序。Cursor 内置 Smart Mode 审批 UI 不经过 Hook，因此部分确认场景无法检测。界面底部有相应提示。

### Q：点击任务能跳转到对应对话吗？

当前版本会尝试打开 transcript 文件。完整的 Cursor 深链跳转计划在后续版本支持。

### Q：如何更新应用？

```bash
cd ~/Projects/cursor-agent-status
git pull   # 若使用 git
./scripts/generate-xcodeproj.sh
xcodebuild -project CursorAgentStatus.xcodeproj \
  -scheme CursorAgentStatus -configuration Release build
```

Hooks 更新：

```bash
./scripts/install-hooks.sh
```

## 9. 卸载与重置

### 卸载 Mac 应用

1. 菜单栏点击图标 → **退出**
2. 删除应用：`rm -rf /Applications/CursorAgentStatus.app`（或你的安装路径）
3. 从「登录项」中移除（若曾设置开机自启）

### 卸载 Hooks

```bash
# 备份后编辑，移除 status-bridge 相关条目
nano ~/.cursor/hooks.json

# 删除 hook 脚本
rm ~/.cursor/hooks/status-bridge.sh
rm ~/.cursor/hooks/status-bridge.py
```

重启 Cursor。

### 清除历史数据

```bash
rm -rf ~/.cursor/agent-status/
```

## 10. 技术说明

### 数据流

```
Cursor Agent 事件
       ↓
~/.cursor/hooks/status-bridge.sh   （归一化 JSON）
       ↓
~/.cursor/agent-status/events.jsonl （事件日志）
       ↓
CursorAgentStatus.app               （FSEvents 实时监听）
       ↓
菜单栏 + 悬浮窗 UI
```

### 监听的事件

| Hook 事件 | 用途 |
|-----------|------|
| `beforeSubmitPrompt` | 用户发送指令（立即显示「处理中」） |
| `afterAgentThought` | Agent 思考中 |
| `sessionStart` / `sessionEnd` | 会话生命周期 |
| `preToolUse` / `postToolUse` / `postToolUseFailure` | 工具调用 |
| `subagentStart` / `subagentStop` | 子代理 |
| `beforeShellExecution` / `afterShellExecution` | Shell 批准 |
| `beforeMCPExecution` / `afterMCPExecution` | MCP 批准 |
| `stop` | Agent 循环结束 |
| `afterAgentResponse` | Agent 回复完成 |

### 相关文件路径

| 路径 | 说明 |
|------|------|
| `~/.cursor/hooks.json` | Hook 配置 |
| `~/.cursor/hooks/status-bridge.sh` | Hook 入口脚本 |
| `~/.cursor/hooks/status-bridge.py` | 事件归一化逻辑 |
| `~/.cursor/agent-status/events.jsonl` | 事件日志 |
| `~/.cursor/agent-status/state.json` | 最新事件快照 |

### 项目源码

```
~/Projects/cursor-agent-status/
├── CursorAgentStatus/    SwiftUI 应用
├── hooks/                Hook 脚本源码
├── scripts/              安装与验证脚本
└── docs/                 文档
```
