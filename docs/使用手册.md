## 1. 这是什么

**Cursor Agent Status** 是一个常驻 macOS 菜单栏的小工具。当你在 Cursor 里使用 Agent（Composer Agent 模式）时，它会在后台监听 Cursor Hooks 上报的事件，并在界面上告诉你：

- 当前有多少任务**正在进行**
- 有哪些操作**等待你确认**
- 哪些任务**刚刚完成**（显示 Hook 返回的 summary）

应用包含两种展示方式：

| 方式 | 说明 |
|------|------|
| **菜单栏图标** | 常驻屏幕右上角，点击展开简要面板 |
| **悬浮 HUD** | Dock 上方的紧凑状态条，每个 Agent 各一个窗口 |

> 应用不会显示在 Dock 栏，这是正常设计（菜单栏专用应用）。

## 2. 系统要求

- **操作系统**：macOS 14.0 或更高
- **Cursor**：已安装并可正常使用 Agent 功能
- **依赖**：Python 3（系统自带即可，用于 Hook 脚本）
- **可选**：Xcode（用于自行编译应用）

## 3. 安装

安装分两步：**先装 Hooks，再启动 Mac 应用**。Hooks 负责采集 Cursor 事件，Mac 应用负责展示。

### 3.1 安装 Hooks（只需一次，更新代码后需重跑）

在终端执行：

```bash
cd ~/Projects/cursor-agent-status
chmod +x scripts/install-hooks.sh hooks/status-bridge.sh hooks/status-bridge.py
./scripts/install-hooks.sh
```

脚本会：

1. 将 `status-bridge.sh` 和 `status-bridge.py` 复制到 `~/.cursor/hooks/`
2. 安装或合并 `~/.cursor/hooks.json`（若已有 hooks 会先备份）
3. 创建数据目录 `~/.cursor/agent-status/`

**安装后请重启 Cursor**，或保存 `~/.cursor/hooks.json` 以触发重新加载。

### 3.2 构建 Mac 应用

**方式 A：Xcode（推荐）**

```bash
open ~/Projects/cursor-agent-status/CursorAgentStatus.xcodeproj
```

在 Xcode 中按 **⌘R** 运行，或 **Product → Archive** 打包安装。

**方式 B：命令行构建**

```bash
cd ~/Projects/cursor-agent-status
xcodebuild -project CursorAgentStatus.xcodeproj \
  -scheme CursorAgentStatus \
  -configuration Debug \
  build
open ~/Library/Developer/Xcode/DerivedData/CursorAgentStatus-*/Build/Products/Debug/CursorAgentStatus.app
```

**方式 C：验证安装是否完整**

```bash
./scripts/verify-integration.sh
```

全部显示 `OK` 即表示 Hooks 和应用均可正常工作。

## 4. 启动应用

### 启动方式

```bash
# Debug 构建产物
open ~/Library/Developer/Xcode/DerivedData/CursorAgentStatus-*/Build/Products/Debug/CursorAgentStatus.app

# 或从「应用程序」文件夹双击打开
```

### 启动后

1. 菜单栏右上角会出现 **✨ / 🔄 / ✋** 图标
2. **不会**出现在 Dock 栏
3. 若已开启「显示悬浮窗」，有 Agent 运行时 Dock 上方会出现 HUD
4. 若图标被折叠，点击菜单栏右侧的 `>>` 查找

### 开机自启（可选）

将 `CursorAgentStatus.app` 拖入：

**系统设置 → 通用 → 登录项 → 打开 at 登录**

## 5. 界面说明

### 5.1 菜单栏图标

| 图标 | 含义 |
|------|------|
| ✨ `sparkles` | 空闲，无进行中的任务 |
| 🔄 `arrow.triangle.2.circlepath` | 有任务正在进行 |
| ✋ `hand.raised` | 有待确认的操作 |

图标右上角**红色数字**表示「进行中」任务数量（最多显示 99）。

### 5.2 菜单栏下拉面板

点击菜单栏图标展开，包含：

**顶部统计卡片**

- **RUN**（蓝色）：当前活跃任务数
- **PND**（橙色）：等待批准或输入的任务数
- **DONE**（绿色）：最近 60 秒内完成的任务数

**三个任务列表**（各最多显示 5 条）

- 刚完成条目显示 Hook **summary** 正文，而非「Agent 任务完成」
- 点击任意任务可尝试打开对应的 transcript 文件

**底部**

| 控件 | 功能 |
|------|------|
| 显示悬浮窗 | 开关，默认开启，持久化保存 |
| 打开 Cursor | 启动 Cursor 应用 |
| 重置 | 清除所有进行中和待确认任务 |
| 退出 | 关闭本应用 |

### 5.3 悬浮 HUD

Dock 上方的紧凑状态条，**每个活跃 Agent 会话各一个窗口**（多 Agent 时水平并排）。

**布局示例**

```
┌─────────────────────────────────────────────┐
│ RUN  需求文档生成                              │
│      正在检查项目状态，准备在 GitHub…  15秒     │  ■
└─────────────────────────────────────────────┘
```

| 区域 | 内容 |
|------|------|
| 状态标签 | `RUN` / `PND` / `DONE`，固定宽度，不会被文字挤压 |
| 第一行 | **Agent 名称**（与 Cursor 侧边栏 Pinned 聊天标题一致） |
| 第二行 | **当前状态** + **步骤耗时**（每秒刷新） |
| 停止按钮 ■ | 进行中时显示，向 Cursor 发送 ⌘⇧⌫ |

**第二行状态来源（优先级）**

| 场景 | 显示内容 |
|------|----------|
| Explore 子代理 | `Exploring · 任务描述` |
| Agent 思考 | `afterAgentThought` 的思考摘要（最长约 80 字） |
| 工具执行中 | 仍优先显示最近一次思考摘要；无思考时显示工具语义（如「修改 Foo.swift」） |
| 等待输入 | `等待确认：{用户指令摘要}` |
| 刚完成 | Hook `summary` 正文 |

**其他特性**

- 宽度自适应 **200px ~ 400px**
- 默认 Dock 上方**水平居中**；可拖动
- 无进行中/待确认任务时自动隐藏所有悬浮窗
- 通过下拉面板 **显示悬浮窗** 开关控制

## 6. 三种任务状态

### 进行中（RUN）

表示 AI 当前正在执行的工作，包括：

| 类型 | 触发时机 | 悬浮窗表现 |
|------|----------|------------|
| 思考 | `afterAgentThought` | 第二行显示思考摘要 |
| Agent 会话 | `sessionStart` | 显示 Agent 名称 |
| 工具执行 | Read、Write、Shell、Grep 等 | 思考摘要或工具语义 + 计时 |
| Subagent | Task / explore 子代理 | `Exploring · …` |

### 待确认（PND）

表示需要你介入的操作：

| 类型 | 说明 |
|------|------|
| 等待用户输入 | Agent 已回复，等待你发送下一条消息 |
| Shell / MCP | 部分批准流程相关状态 |
| 操作被拒绝 | 权限拒绝后需重新确认 |

> **注意**：Cursor Smart Mode 内置审批不一定能被 Hook 捕获。

### 刚完成（DONE）

最近 **60 秒内**结束、且 Hook 提供了 **summary** 的任务。

- 菜单栏「刚完成」列表显示 summary 正文
- 无 summary 的完成事件不会写入列表
- 60 秒后自动淡出

## 7. 日常使用

### 典型工作流

```
1. 启动 Cursor Agent Status
2. 在 Cursor 中打开一个或多个 Agent，发送任务
3. 菜单栏角标增加 → Dock 上方出现对应数量的悬浮窗
4. 观察第二行：思考摘要、Exploring、步骤计时
5. 需要停止时点击悬浮窗右侧 ■
6. 点击菜单栏图标查看 RUN / PND / DONE 详情
```

### 更新代码后的标准流程

```bash
cd ~/Projects/cursor-agent-status
git pull                          # 若使用 git
./scripts/install-hooks.sh        # 同步 Hook 脚本
xcodebuild -scheme CursorAgentStatus -configuration Debug build
pkill -x CursorAgentStatus
open DerivedData/.../CursorAgentStatus.app
# 建议重启 Cursor
```

### 自动过期机制

| 类型 | 过期时间 |
|------|----------|
| 工具 / Subagent | 90 秒无更新 |
| Shell/MCP 待确认 | 60 秒无更新 |
| 会话 | 3 分钟无任何新事件 |
| 刚完成 | 60 秒后淡出 |

### 手动重置

若重启 Cursor 后仍看到旧的进行中任务，点击 **「重置」** 即可立即清空。

## 8. 常见问题

### Q：菜单栏找不到图标？

- 检查应用是否在运行：`pgrep -l CursorAgentStatus`
- 查看菜单栏右侧 `>>` 折叠区
- 重新启动应用

### Q：Cursor 里 Agent 在工作，但应用显示 0？

1. 确认 Hooks 已安装：`grep status-bridge ~/.cursor/hooks.json`
2. 重启 Cursor
3. 检查事件文件：
   ```bash
   tail -f ~/.cursor/agent-status/events.jsonl
   ```
4. 重启 Cursor Agent Status 应用

### Q：悬浮窗没有显示思考内容？

Hook 可能已写入但 App 未更新：

```bash
grep afterAgentThought ~/.cursor/agent-status/events.jsonl | tail -1
```

若有输出但仍不显示：

1. `./scripts/install-hooks.sh`
2. 重新编译并启动 App
3. 重启 Cursor

### Q：Agent 名称显示为「Agent」而不是侧边栏标题？

App 从 Cursor 状态库读取聊天标题。若读取失败则回退为 `Agent` 或项目文件夹名。请确认：

- Cursor 已正常运行且该聊天在侧边栏有标题
- 使用的是最新版 App（含 `ComposerNameResolver`）

### Q：开了两个 Agent 只看到一个悬浮窗？

- 确认 App 为最新编译版本（多窗功能）
- 只有标准 UUID 格式的会话会计入（测试/demo 会话会被忽略）
- 某一 Agent 若已结束（无 RUN/PND）则不会显示窗口

### Q：Cloud Agent 能显示吗？

**不能。** 用户级 Hooks 仅作用于本地 Cursor IDE。

### Q：点击任务能跳转到对应对话吗？

当前会尝试打开 transcript 文件，尚不支持 Cursor 深链。

## 9. 卸载与重置

### 卸载 Mac 应用

1. 菜单栏 → **退出**
2. 删除 `CursorAgentStatus.app`
3. 从「登录项」中移除（若曾设置）

### 卸载 Hooks

编辑 `~/.cursor/hooks.json` 移除 status-bridge 条目，然后：

```bash
rm ~/.cursor/hooks/status-bridge.sh
rm ~/.cursor/hooks/status-bridge.py
```

重启 Cursor。

### 清除历史数据

```bash
rm -rf ~/.cursor/agent-status/
```

## 10. 技术说明

### 数据流

```
Cursor Agent 事件
       ↓
~/.cursor/hooks/status-bridge.sh
       ↓
~/.cursor/hooks/status-bridge.py   （归一化 JSON）
       ↓
~/.cursor/agent-status/events.jsonl
       ↓
CursorAgentStatus.app
       ├── StatusStore              （状态机 + 思考摘要缓存）
       ├── ComposerNameResolver     （读取 Cursor 聊天标题）
       └── FloatingPanelController  （按 conversationId 多窗）
```

### 监听的事件

| Hook 事件 | 用途 |
|-----------|------|
| `beforeSubmitPrompt` | 用户指令摘要 |
| `afterAgentThought` | 思考摘要（悬浮窗优先展示） |
| `sessionStart` / `sessionEnd` | 会话生命周期 |
| `preToolUse` / `postToolUse` / `postToolUseFailure` | 工具调用 |
| `subagentStart` / `subagentStop` | 子代理 |
| `beforeShellExecution` / `afterShellExecution` | Shell |
| `beforeMCPExecution` / `afterMCPExecution` | MCP |
| `stop` | 任务结束（summary） |
| `afterAgentResponse` | Agent 回复 |

### 相关文件路径

| 路径 | 说明 |
|------|------|
| `~/.cursor/hooks.json` | Hook 配置 |
| `~/.cursor/hooks/status-bridge.py` | 事件归一化 |
| `~/.cursor/agent-status/events.jsonl` | 事件日志 |
| `~/Library/Application Support/Cursor/User/globalStorage/state.vscdb` | Agent 聊天标题 |

### 项目源码

```
~/Projects/cursor-agent-status/
├── CursorAgentStatus/
│   ├── Controllers/FloatingPanelController.swift
│   ├── Services/ComposerNameResolver.swift
│   ├── Services/StatusStore.swift
│   └── Views/FloatingPanelView.swift
├── hooks/
├── scripts/
└── docs/
```
